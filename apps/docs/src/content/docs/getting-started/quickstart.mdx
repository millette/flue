**Flue** is a TypeScript framework for building agent-backed applications. Define an agent with a model and instructions, then run it locally or deploy it as part of your application. Flue handles the runtime around model calls, including continuing conversations and controlled execution, with targets for Node.js and Cloudflare.

## Prerequisites

- **Node.js `>=22.18.0`** — required by `@flue/runtime` and `@flue/cli`.
- **An LLM provider** — provide credentials for your chosen model. On Cloudflare, you can instead configure a Workers AI binding and use a model such as `cloudflare/@cf/moonshotai/kimi-k2.6`.
- **A coding agent (recommended)** — use one to run the Quickstart prompt and help extend your agent later.
- **A sandbox provider (optional)** — Flue includes a built-in virtual sandbox. Connect a [supported sandbox](/docs/ecosystem/overview/#sandbox-connectors) if your agent needs a more capable execution environment, such as access to shell tools.

## Manual installation

***Note:** We recommend the [Quickstart](#quickstart) prompt above for guided setup and automatic validation. Use the steps below if you prefer to create your first agent by hand.*

### 1. Install Flue

In a new directory, install Flue and initialize your target. The `flue init` command creates `flue.config.ts`; you will add an agent module in the next step.

```bash
npm install @flue/runtime
npm install --save-dev @flue/cli
npx flue init --target node # or: --target cloudflare
```

### 2. Create your first agent module

Create `agents/assistant.ts`. This example uses Claude Sonnet, but you can choose any [supported model](https://pi.dev/models) and configure its provider credentials instead:

```ts title="agents/assistant.ts"
import { createAgent } from '@flue/runtime';

export default createAgent(() => ({
  model: 'anthropic/claude-sonnet-4-6',
  instructions: 'Help the user clearly and concisely.',
}));
```

Exporting the agent from `agents/assistant.ts` makes it available in Flue as `assistant`.

### 3. Provide LLM credentials

The example agent uses Anthropic, so create a local `.env` file with your Anthropic API key:

```dotenv title=".env"
ANTHROPIC_API_KEY="your-api-key"
```

Add `.env` to `.gitignore` and do not commit provider credentials. If you chose another model in step 2, set the API key required by its provider instead.

### 4. Talk to your agent locally

For a Node.js target, open an interactive session with the discovered agent:

```bash
npx flue connect assistant local --env .env
```

Once connected, type a message and press Enter. In this command, `assistant` is the module name and `local` is the ID for this agent instance.

```text
$ npx flue connect assistant local --env .env
[flue] Connected to assistant/local. Enter a prompt per line; Ctrl-D to exit.
Hello! What can you help me with?
{
  "text": "I can help answer questions, draft content, summarize information, and more."
}
```

Congratulations! You have created your first Flue agent and opened a local conversation with it. From here, you can shape its behavior, add capabilities, or deploy it as part of an application.
